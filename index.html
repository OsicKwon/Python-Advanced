Getting Started

Getting started with this Jekyll site is straightforward. Follow these steps to set up your local development environment and start creating content.

Prerequisites

Before you begin, make sure you have the necessary tools installed on your system. These are essential for running Jekyll locally.

Ruby Installation

Jekyll is built on Ruby, so you'll need Ruby installed on your system. Most modern operating systems come with Ruby pre-installed, but you may need to install a specific version depending on your needs.

RubyGems Package Manager

RubyGems is Ruby's package manager, used to install Jekyll and other Ruby gems. It typically comes bundled with Ruby installations.

Bundler

Bundler manages your project's dependencies and ensures consistent gem versions across different environments. Install it using: gem install bundler

Installation Steps

Once you have the prerequisites installed, follow these steps to get your Jekyll site up and running.

Step 1: Install Dependencies

Navigate to your project directory and run bundle install. This will install all the required gems specified in your Gemfile, including Jekyll and its plugins.

Step 2: Start the Development Server

Run ./serve.sh or bundle exec jekyll serve to start the local development server. Jekyll will build your site and make it available at http://localhost:4000.

Step 3: View Your Site

Open your web browser and navigate to http://localhost:4000. You should see your Jekyll site with all the tocbot functionality working. Any changes you make to your content will automatically regenerate the site.

Creating Your First Post

Now that your site is running, you can start creating content. Jekyll makes it easy to add new blog posts and pages.

Post File Structure

Blog posts should be placed in the _posts directory with a filename following the format: YYYY-MM-DD-title.md. The date prefix is important for Jekyll to recognize and process the post correctly.

Front Matter

Each post needs front matter at the top, enclosed in triple dashes. This YAML front matter contains metadata about the post, such as title, date, and layout.

Writing Content

Write your content in Markdown below the front matter. Use headings to structure your content, and tocbot will automatically generate a table of contents from these headings.

About Tocbot

Tocbot is a lightweight JavaScript library that automatically generates a table of contents from the headings on your page. It's designed to be simple, fast, and highly customizable.

How Tocbot Works

Tocbot scans your page for headings (h1 through h6) and creates a hierarchical table of contents. It then provides smooth scrolling navigation and highlights the currently active section as you scroll through the content.

Automatic Detection

Tocbot automatically detects all headings within a specified container. You can configure which heading levels to include and how the table of contents should be structured.

Dynamic Updates

As you scroll through the page, tocbot dynamically updates the active link in the table of contents, making it easy to see which section you're currently reading.

Smooth Scrolling

When you click a link in the table of contents, tocbot smoothly scrolls to that section. The scroll offset can be customized to account for fixed headers or other page elements.

Tocbot Configuration

Tocbot offers extensive configuration options that allow you to customize its behavior to match your site's needs.

Heading Selectors

You can specify which heading levels to include in the table of contents. For example, you might only want h2 and h3 headings, or you might want all heading levels from h1 to h6.

Scroll Offset

The scroll offset determines how far from the top of the viewport the target heading should be positioned when clicked. This is useful when you have a fixed header that would otherwise cover the heading.

Collapsible Sections

Tocbot can be configured to create collapsible sections in the table of contents, allowing users to expand or collapse subsections as needed.

Content Structure

Understanding how to structure your content is crucial for getting the most out of tocbot. Well-structured content with proper heading hierarchy makes navigation intuitive and effective.

Heading Hierarchy

Maintain a logical heading hierarchy in your content. Start with h1 for the main title, use h2 for major sections, h3 for subsections, and so on. Don't skip heading levels (e.g., don't go from h2 directly to h4).

Best Practices

Follow these best practices when structuring your content with headings:

  • Use only one h1 per page
  • Maintain consistent heading levels
  • Use descriptive heading text
  • Keep heading text concise but meaningful

Content Organization

Organize your content into logical sections that flow naturally. Each section should have a clear purpose and contribute to the overall narrative or information architecture of the page.

Section Length

While there's no hard rule about section length, aim for sections that are substantial enough to provide value but not so long that they become overwhelming. Break up very long sections into smaller subsections.

Visual Hierarchy

Use heading levels to create a visual hierarchy that guides readers through your content. Major topics should use higher-level headings, while subtopics use lower-level headings.
